⭐ First Missing Positive

Cyclic Sort + Scan — LeetCode 041

📖 Problema

Dado um array de inteiros (pode conter negativos, zeros, duplicatas), encontre o menor inteiro positivo ausente. Deve ser O(n) tempo e O(1) espaço.

Teste: FirstMissingPositiveTest()  →  entrada [3, 4, -1, 1], esperado 2

Ideia: Cyclic Sort — colocar cada número positivo k na posição k−1 (ignorar negativos e k ≥ n). Depois varrer: primeiro índice i onde nums[i] ≠ i+1 dá a resposta i+1.

🔍 Passos do Algoritmo

1Fase 1 — Cyclic Sort: Para cada posição i, se nums[i] é positivo, menor que n e não está na posição correta, trocar com seu destino.
2i=0, temp=3: 3 deve estar em pos 2. nums[2]=−1 ≠ 3 → trocar pos 0 ↔ 2. Array: [−1, 4, 3, 1]. temp=−1 ≤ 0, parar.
3i=1, temp=4: condição 4 < 4 é falsa (fora do range) → nenhuma troca.
4i=2, temp=3: nums[2]=3 == temp → já está correto → nenhuma troca.
5i=3, temp=1: 1 deve estar em pos 0. nums[0]=−1 ≠ 1 → trocar pos 3 ↔ 0. Array: [1, 4, 3, −1]. temp=−1 ≤ 0, parar.
6Fase 2 — Scan: [1, 4, 3, −1]. i=0: 1==1 ✓. i=1: 4≠2 ✗ → retornar i+1 = 2.

⏱ Tempo

O(n) — cyclic sort: cada elemento é movido no máximo uma vez. Scan: O(n). Total: O(n).

💾 Espaço

O(1) — ordenação in-place, sem estruturas auxiliares.

💻 C#

public int FirstMissingPositive(int[] nums)
{
    int i, temp;
    // Fase 1: Cyclic Sort
    for (i = 0; i < nums.Length; i++)
    {
        temp = nums[i];
        while (temp > 0 && temp < nums.Length && nums[temp - 1] != temp)
        {
            nums[i] = nums[temp - 1];
            nums[temp - 1] = temp;
            temp = nums[i];
        }
    }
    // Fase 2: Scan
    for (i = 0; i < nums.Length; i++)
        if (nums[i] != i + 1) break;
    return i + 1;
}
