➕ Two Sum II — Input Array Is Sorted

Two Pointers — LeetCode Problema 0167

📚 Qual e o Problema?

Dado um array ordenado numbers e um inteiro target, retorne os indices (1-based) dos dois numeros que somam exatamente target. Garantido que existe exatamente uma solucao.

Este visualizador usa o teste TwoSumTest_1:
numbers = [2, 7, 11, 15], target = 9  →  Resultado esperado: [1, 2]  (2 + 7 = 9)

A solucao usa Two Pointers: i começa no menor elemento (esquerda) e j no maior (direita). Se a soma for igual ao target, achou! Se for maior, recua j. Se for menor, avanca i. Isso funciona porque o array esta ordenado.

🔍 Passos do Algoritmo

1 Inicializacao: i = 0 (menor elemento), j = n - 1 (maior elemento)
2 Calcular soma: compare = (numbers[i] + numbers[j]).CompareTo(target)
3 ✅ Encontrado! compare == 0 → retorna { i+1, j+1 } (indices 1-based)
4 Soma muito grande: compare > 0 → j-- (reduce o maior valor)
5 Soma muito pequena: compare < 0 → i++ (aumenta o menor valor)
6 Sem solucao: i ≥ j → retorna [] (array vazio)

⏱️ Complexidade de Tempo

O(n) — Linear

Cada elemento e visitado no maximo uma vez: i so avanca, j so recua. O array ordenado garante que a convergencia dos ponteiros sempre encontra a solucao.

💾 Complexidade de Espaco

O(1) — Constante

Apenas dois ponteiros: i e j. Sem estruturas de dados auxiliares (diferente da abordagem com HashMap do Two Sum I).

💻 Implementacao em C#

public int[] TwoSum(int[] numbers, int target)
{
    int i = 0, j = numbers.Length - 1;

    while (i < j)
    {
        int compare = (numbers[i] + numbers[j]).CompareTo(target);

        if      (compare == 0) return new int[] { i + 1, j + 1 }; // ✅ achou!
        else if (compare >  0) j--;   // soma muito grande → recua direita
        else if (compare <  0) i++;   // soma muito pequena → avanca esquerda
    }

    return new int[] { };           // sem solucao
}
